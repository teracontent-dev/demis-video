# 웹 강의영상 자동화 프로젝트 — 최종 설계 (v3)

> **결정 사항**
> - 배포: 웹 서버 (로컬 먼저, 추후 EC2)
> - 스크립트↔슬라이드: AI 자동 생성 + 채팅 기반 수정 (Claude Opus 4.6 고정)
> - 렌더링: Puppeteer + FFmpeg 커스텀
> - 폰트: Pretendard Variable (CDN)
> - 디자인: `tds-design-system` 스킬 참조 (TailwindCSS 금지, CSS Variables + CSS Modules)
> - 뷰포트: 1920×1080 풀스크린 고정
> - **에디터 UI: React 18 + Vite + TipTap + Zustand** (편집 도구)
> - **슬라이드 엔진: 순수 바닐라 JS** (영상 결과물, iframe 독립 — React 종속 없음)

### 1-1. 아키텍처 핵심 원칙: 에디터 vs 엔진 완전 분리

```
⚠️ 이 프로젝트는 두 개의 완전히 독립된 웹 앱으로 구성된다.
   절대 섞지 말 것.

┌─ 에디터 (React) ─────────────────────────────────────┐
│ 역할: 편집 도구 UI                                    │
│ 기술: React 18 + Vite + TypeScript                    │
│ 라이브러리: TipTap (블록에디터), Zustand (상태관리)     │
│ 디자인: TDS 스킬 참조, CSS Modules                     │
│ 포함: 패널, 버튼, 채팅창, 타임라인, 슬라이드 목록      │
│                                                       │
│ ※ 에디터는 "도구"일 뿐. 영상 결과물에 포함되지 않음.   │
│                                                       │
│   ┌─ iframe ─────────────────────────────────────┐    │
│   │                                               │    │
│   │  엔진 (바닐라 JS) ← React 전혀 모름           │    │
│   │  역할: 슬라이드 렌더링 (= 영상의 실제 화면)   │    │
│   │  기술: 순수 JS + CSS + Web Animations API     │    │
│   │  트랜지션: fade, slide, wipe, dissolve, morph │    │
│   │  애니메이션: typewriter, fadeIn, slideUp ...   │    │
│   │  API: renderFrame(timeMs) 하나만 노출         │    │
│   │                                               │    │
│   │  ※ Puppeteer가 이것만 열어서 프레임 캡처      │    │
│   │  ※ 영상 결과물 = 이 엔진의 출력 그 자체       │    │
│   │  ※ React, TipTap, Zustand 일절 포함 안 됨     │    │
│   │                                               │    │
│   └───────────────────────────────────────────────┘    │
│         ↕ postMessage 통신만 (JSON 데이터만 오감)      │
└───────────────────────────────────────────────────────┘
```

**통신 규칙:**
- 에디터 → 엔진: `postMessage`로 슬라이드 JSON 전달
- 엔진 → 에디터: `postMessage`로 재생 시간/상태 전달
- 두 앱 사이에 import, require, 공유 모듈 없음
- 엔진은 `slide-engine/` 폴더에 완전 독립 존재
- 엔진의 `index.html`을 브라우저에서 직접 열어도 동작해야 함

**왜 이렇게 분리하는가:**
- 슬라이드 엔진에 React가 들어가면 Puppeteer 렌더링 시 불필요한 프레임워크 오버헤드 발생
- 트랜지션/애니메이션은 순수 CSS + JS가 가장 정밀한 프레임 제어 가능
- 엔진 독립 = 나중에 에디터를 Electron이든 다른 프레임워크든 교체해도 엔진은 그대로

## 3. 프로젝트 데이터 스키마

```typescript
// === 프로젝트 최상위 ===
interface Project {
  id: string;
  title: string;
  createdAt: string;
  updatedAt: string;
  settings: ProjectSettings;
  script: TipTapDocument;         // TipTap JSON
  slides: Slide[];                // AI가 생성/수정
  timeline: Timeline;
  chatHistory: ChatMessage[];
}

interface ProjectSettings {
  resolution: [1920, 1080];       // 고정
  fps: 30;
  fontFamily: "Pretendard Variable";
  exportPreset: "draft" | "standard" | "high" | "ultra";
}

// === 슬라이드 ===
interface Slide {
  id: string;
  order: number;
  duration: number;               // ms
  background: {
    type: "solid" | "gradient" | "image";
    value: string;
  };
  elements: SlideElement[];
  transition: {
    type: TransitionType;
    duration: number;             // ms
    easing: string;               // CSS easing
  };
  notes: string;
}

interface SlideElement {
  id: string;
  type: "text" | "image" | "video" | "shape";
  position: {
    x: number;       // 0~100 (% 비율, 좌측 기준)
    y: number;       // 0~100 (% 비율, 상단 기준)
    width: number;   // 0~100 (% 비율)
    height: number;  // 0~100 (% 비율)
  };
  // ⚠️ 좌표계는 비율(%) 기반. 절대 px 사용 금지.
  // 예: { x: 10, y: 20, width: 80, height: 15 }
  //   → 1080p: x=192px, y=216px, w=1536px, h=162px
  //   → 4K:    x=384px, y=432px, w=3072px, h=324px
  // 엔진이 렌더링 시 해상도에 맞게 자동 변환함.
  rotation: number;               // degrees
  opacity: number;                // 0~1
  zIndex: number;
  style: Record<string, any>;
  content: string | MediaRef;
  animation: {
    enter: AnimationConfig;
    exit?: AnimationConfig;
    emphasis?: AnimationConfig;
  };
}

interface AnimationConfig {
  type: string;                   // "fadeIn" | "slideUp" | "typewriter" ...
  duration: number;               // ms
  delay: number;                  // ms
  easing: string;
}

type TransitionType =
  | "none" | "fade" | "slide-left" | "slide-right" | "slide-up"
  | "wipe-left" | "wipe-right" | "scale" | "dissolve" | "morph";

// === 타임라인 ===
interface Timeline {
  totalDuration: number;          // ms (슬라이드 합산)
  audioTracks: AudioTrack[];
  subtitleTrack: SubtitleCue[];
}

interface AudioTrack {
  id: string;
  type: "tts" | "bgm" | "recording";
  src: string;                    // 파일 경로
  startTime: number;              // ms
  duration: number;
  volume: number;                 // 0~1
}

interface SubtitleCue {
  id: string;
  startTime: number;
  endTime: number;
  text: string;
}

// === AI 채팅 ===
interface ChatMessage {
  id: string;
  role: "user" | "assistant";
  content: string;
  timestamp: string;
  slideChanges?: {
    changedIndices: number[];
    previousState: Slide[];       // Undo 스택
    diff: Record<string, any>;
  };
}

// === 미디어 참조 ===
interface MediaRef {
  id: string;
  type: "image" | "video";
  src: string;                    // /api/media/{id}
  thumbnail?: string;
  originalName: string;
}
```

## 4. AI 채팅 기반 슬라이드 편집

### 4-1. 워크플로우

```
사용자: 스크립트 작성 완료
  → POST /api/ai/generate-slides { script_text, style_preset }
  ← { slides[], timeline }
  → 프리뷰에 즉시 렌더링

사용자: "3번 슬라이드 배경을 어둡게"
  → POST /api/ai/chat-edit {
      project_id,
      message,
      context: { current_slides, selected_slide_index, chat_history }
    }
  ← {
      updated_slides[],
      changed_indices: [2],
      ai_message: "3번 슬라이드 배경을 #191f28로 변경했어요.",
      diff: { ... }
    }
  → 변경된 슬라이드만 프리뷰 업데이트
  → [되돌리기] 버튼에 이전 상태 저장
```

### 4-2. AI 모델 & 프롬프트 시스템

> **⚠️ 슬라이드 디자인 두뇌는 `claude-opus-4-6` 고정.**
> 스크립트→슬라이드 생성, 채팅 수정 모두 Opus 4.6만 사용.
> 다른 모델(Sonnet, Haiku, Gemini 등)로 대체하지 말 것.
> 디자인 판단, 레이아웃 결정, 색상 선택, 애니메이션 설계 모두 이 모델이 담당.

```python
# ai_service.py
import anthropic
import os

client = anthropic.Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))

# ⭐ 슬라이드 디자인 모델 — 절대 변경 금지
SLIDE_DESIGN_MODEL = "claude-opus-4-6"

SYSTEM_PROMPT = """
당신은 강의 슬라이드 디자인 전문가입니다.
스크립트나 수정 요청을 받아 슬라이드 JSON을 생성/수정합니다.

## 슬라이드 JSON 스키마
(TypeScript 인터페이스 Slide, SlideElement 그대로 포함)

## 디자인 원칙
- Pretendard Variable 폰트 (400 Regular, 600 SemiBold, 700 Bold)
- 색상은 TDS 컬러 토큰 사용 (grey900 텍스트, blue500 강조 등)
- 좌표계: 1920×1080px 기준
- 한 슬라이드에 핵심 메시지 하나
- 여백 충분히

## 수정 규칙
- 요청된 부분만 변경, 나머지 유지
- changed_indices로 변경 위치 명시
- 모호한 요청은 합리적 해석 후 ai_message로 근거 설명
"""

async def generate_slides(script_text: str, style_preset: str = "modern") -> dict:
    """스크립트 → 슬라이드 자동 생성 (Opus 4.6 고정)"""
    response = client.messages.create(
        model=SLIDE_DESIGN_MODEL,  # claude-opus-4-6 고정
        max_tokens=8192,
        system=SYSTEM_PROMPT,
        messages=[{
            "role": "user",
            "content": f"다음 스크립트를 분석해서 슬라이드 JSON을 생성해주세요.\n\n{script_text}"
        }]
    )
    return parse_slide_response(response)

async def chat_edit(message: str, context: dict) -> dict:
    """채팅으로 슬라이드 수정 (Opus 4.6 고정)"""
    response = client.messages.create(
        model=SLIDE_DESIGN_MODEL,  # claude-opus-4-6 고정
        max_tokens=8192,
        system=SYSTEM_PROMPT,
        messages=[
            *context["chat_history"],
            {
                "role": "user",
                "content": f"""현재 슬라이드 상태:
{json.dumps(context["current_slides"], ensure_ascii=False)}

선택된 슬라이드: {context.get("selected_slide_index", "없음")}

수정 요청: {message}"""
            }
        ]
    )
    return parse_edit_response(response)
```

### 5-2. 좌표 시스템 (비율 기반)

```
⚠️ 모든 SlideElement의 position은 % 비율 (0~100).
   엔진이 현재 해상도에 맞게 px로 자동 변환.

JSON 데이터:  { x: 10, y: 20, width: 80, height: 15 }
                 ↓ 엔진이 해상도에 맞게 변환
  1080p (1920×1080):  x=192px,  y=216px,  w=1536px, h=162px
  4K    (3840×2160):  x=384px,  y=432px,  w=3072px, h=324px

변환 공식:
  px_x = (position.x / 100) * canvas.width
  px_y = (position.y / 100) * canvas.height
```

```javascript
// element-renderer.js
function toPixel(position, canvasWidth, canvasHeight) {
  return {
    x: (position.x / 100) * canvasWidth,
    y: (position.y / 100) * canvasHeight,
    width: (position.width / 100) * canvasWidth,
    height: (position.height / 100) * canvasHeight,
  };
}
```

폰트 크기도 동일하게 비율 적용:
- JSON에 `fontSize: 2.5` → 캔버스 높이의 2.5%
- 1080p: `2.5% × 1080 = 27px`
- 4K: `2.5% × 2160 = 54px`

### 5-3. 내보내기 흐름 (사용자 관점)

### 5-5. 프리뷰 시스템 (편집 중 실시간 재생)

> **편집 화면에서 [▶ 재생] 누르면 슬라이드 + 내레이션 + 자막이 싱크되어 재생된다.**
> 렌더링(내보내기) 전에 결과물을 미리 확인하는 기능.

```
┌─────────────────────────────────────┐
│        슬라이드 프리뷰 영역          │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  │   iframe (슬라이드 엔진)     │    │
│  │   슬라이드 화면 + 애니메이션 │    │
│  │   + 자막 오버레이           │    │
│  │                             │    │
│  │   ┌───────────────────────┐ │    │
│  │   │ 오늘은 인공지능의 역사 │ │ ← 자막 (실시간 표시)
│  │   └───────────────────────┘ │    │
│  └─────────────────────────────┘    │
│                                     │
│  [◀prev] [▶재생/⏸정지] [▶next]     │ ← 컨트롤 바
│  ▓▓▓▓▓▓▓▓░░░░░░░░░░░ 00:23/02:15  │ ← 시크바 (클릭/드래그)
│  🔊━━━━━━━━━╸          [1x▾]       │ ← 볼륨 + 배속
└─────────────────────────────────────┘
```

**프리뷰에서 동시에 재생되는 것들:**
1. 슬라이드 화면 (트랜지션 + 엘리먼트 애니메이션)
2. 내레이션 오디오 (ElevenLabs TTS 결과물)
3. 자막 (오디오 타임스탬프에 싱크)

**프리뷰 동작 원리:**

```
[▶ 재생] 클릭
  ↓
React (에디터)
  ├─ postMessage("PLAY") → iframe (슬라이드 엔진)
  ├─ audioElement.play() → 내레이션 오디오 재생
  └─ startSubtitleSync() → 자막 타이머 시작
  ↓
매 프레임 (requestAnimationFrame):
  ├─ 현재 시간 = audioElement.currentTime (오디오가 마스터 클럭)
  ├─ postMessage("SEEK", currentTime) → 슬라이드 엔진 프레임 동기화
  └─ updateSubtitle(currentTime) → 해당 시간의 자막 표시
```

**핵심: 오디오가 마스터 클럭.** 슬라이드와 자막은 오디오 시간에 맞춰 따라감.
오디오가 없는 슬라이드는 내부 타이머(requestAnimationFrame)가 마스터.

### 5-6. 프리뷰 컨트롤 상세

```typescript
// hooks/usePreviewPlayer.ts

interface PreviewPlayer {
  // 상태
  isPlaying: boolean;
  currentTime: number;        // ms
  totalDuration: number;      // ms
  volume: number;             // 0~1
  playbackRate: number;       // 0.5, 1, 1.5, 2

  // 액션
  play: () => void;
  pause: () => void;
  seek: (timeMs: number) => void;
  setVolume: (v: number) => void;
  setPlaybackRate: (rate: number) => void;
  prevSlide: () => void;
  nextSlide: () => void;
}

function usePreviewPlayer(iframeRef, audioRef) {
  const { slides, timeline } = useProjectStore();
  const [isPlaying, setIsPlaying] = useState(false);
  const [currentTime, setCurrentTime] = useState(0);
  const animFrameRef = useRef<number>();

  const play = () => {
    setIsPlaying(true);

    // 내레이션 오디오 재생
    if (audioRef.current) {
      audioRef.current.currentTime = currentTime / 1000;
      audioRef.current.play();
    }

    // 슬라이드 엔진에 PLAY 전송
    iframeRef.current?.contentWindow?.postMessage({ type: "PLAY" }, "*");

    // 동기화 루프 시작
    const syncLoop = () => {
      let time;
      if (audioRef.current && !audioRef.current.paused) {
        time = audioRef.current.currentTime * 1000; // 오디오가 마스터
      } else {
        time = currentTime + (1000 / 60); // 오디오 없으면 프레임 기반
      }
      setCurrentTime(time);

      // 슬라이드 엔진 동기화
      iframeRef.current?.contentWindow?.postMessage({
        type: "SEEK", payload: { timeMs: time }
      }, "*");

      animFrameRef.current = requestAnimationFrame(syncLoop);
    };
    syncLoop();
  };

  const pause = () => {
    setIsPlaying(false);
    audioRef.current?.pause();
    iframeRef.current?.contentWindow?.postMessage({ type: "PAUSE" }, "*");
    cancelAnimationFrame(animFrameRef.current);
  };

  const seek = (timeMs: number) => {
    setCurrentTime(timeMs);
    if (audioRef.current) audioRef.current.currentTime = timeMs / 1000;
    iframeRef.current?.contentWindow?.postMessage({
      type: "SEEK", payload: { timeMs }
    }, "*");
  };

  // ... volume, playbackRate, prevSlide, nextSlide
}
```

### 6-6. TTS 내레이션 + 자막 + 타임라인 시스템

> **⚠️ 핵심 원칙: 타임라인이 마스터. 슬라이드/오디오/자막은 타임라인 위의 독립 트랙.**
> 각 트랙은 자기만의 startTime과 duration을 가짐.
> 음성이 있는 슬라이드는 음성에 맞추고, 음성이 없는 슬라이드는 자유롭게 길이 설정.

```
타임라인 (마스터)
│
├─ 슬라이드 트랙  [인트로 3s][목차 5s][내용1 4.2s][내용2 6.8s][마무리 3s]
├─ 오디오 트랙    [───없음───][없음──][음성1─────][음성2──────][없음────]
└─ 자막 트랙      [───없음───][없음──][자막1─────][자막2──────][없음────]
                                      ↑                       ↑
                              음성 있는 구간:             음성 없는 구간:
                              duration = 음성길이+여유    duration = 수동 설정

예시: 슬라이드 5개, 음성은 3번 4번만 있는 경우
  슬라이드1 (인트로):  0초~3초       ← 음성 없음, duration 수동 3초
  슬라이드2 (목차):    3초~8초       ← 음성 없음, duration 수동 5초
  슬라이드3 (내용1):   8초~12.2초    ← 음성1 재생 (4.2초)
  슬라이드4 (내용2):   12.2초~19초   ← 음성2 재생 (6.8초)
  슬라이드5 (마무리):  19초~22초     ← 음성 없음, duration 수동 3초
```

**두 가지 모드가 공존:**

```
모드 A: 내레이션 → 슬라이드 맞춤 (자동)
  스크립트 작성 → [내레이션 생성] 클릭
  → 음성 생성 → 음성 길이에 맞춰 slide.duration 자동 설정
  → 자막은 음성 타임스탬프에서 자동 추출
  → 음성이 왕, 슬라이드/자막이 따라감

모드 B: 슬라이드 먼저 → 음성 나중에 배치 (수동)
  슬라이드 먼저 만들어놓고 → 중간에 목차/인트로 삽입
  → 음성이 없는 슬라이드는 duration 수동 설정
  → 기존 음성의 startTime만 밀림 (음성 자체는 재생성 안 함)
  → 자막도 startTime만 밀림

두 모드 혼합 가능:
  내레이션 생성된 상태에서 중간에 목차 삽입
  → 목차 슬라이드: duration 수동 5초, 음성 없음
  → 뒤에 있던 음성/자막의 startTime이 자동으로 5초 밀림
  → 음성 재생성 없음!
```

**슬라이드 삽입/삭제/이동 시 타임라인 자동 재계산:**

```python
def recalculate_timeline(slides: list, audio_tracks: list, subtitle_cues: list) -> dict:
    """
    슬라이드 순서가 변경되면 오디오/자막의 startTime을 자동 재배치.
    ⭐ 음성 파일 자체는 건드리지 않음. startTime만 밀거나 당김.
    """
    cumulative_time = 0
    updated_audio = []
    updated_cues = []

    for slide in slides:
        slide_start = cumulative_time
        slide_duration = slide["duration"]

        # 이 슬라이드에 연결된 오디오 트랙 찾기
        linked_audio = next(
            (a for a in audio_tracks if a["linkedSlideId"] == slide["id"]),
            None
        )

        if linked_audio:
            # 오디오 startTime을 슬라이드 시작에 맞춤
            linked_audio["startTime"] = slide_start

            # 이 오디오에 연결된 자막도 offset 재계산
            original_offset = linked_audio.get("_originalStartTime", 0)
            time_shift = slide_start - original_offset

            for cue in subtitle_cues:
                if cue.get("linkedAudioId") == linked_audio["id"]:
                    cue["startTime"] = cue["_originalStartTime"] + time_shift
                    cue["endTime"] = cue["_originalEndTime"] + time_shift

            updated_audio.append(linked_audio)

        cumulative_time += slide_duration

    return {
        "total_duration": cumulative_time,
        "audio_tracks": updated_audio,
        "subtitle_cues": [c for c in subtitle_cues if c.get("startTime") is not None],
    }
```

**내레이션 생성 코드:**

```python
# tts_service.py
import httpx, os, json, base64, hashlib, asyncio

ELEVENLABS_API_KEY = os.getenv("ELEVENLABS_API_KEY")
ELEVENLABS_BASE = "https://api.elevenlabs.io/v1"

def _text_hash(text: str) -> str:
    return hashlib.md5(text.encode()).hexdigest()[:12]

async def generate_narration(
    text: str,
    voice_id: str = "21m00Tcm4TlvDq8ikWAM",
    model_id: str = "eleven_multilingual_v2",
) -> dict:
    async with httpx.AsyncClient() as client:
        response = await client.post(
            f"{ELEVENLABS_BASE}/text-to-speech/{voice_id}/with-timestamps",
            headers={
                "xi-api-key": ELEVENLABS_API_KEY,
                "Content-Type": "application/json",
            },
            json={
                "text": text,
                "model_id": model_id,
                "voice_settings": { "stability": 0.5, "similarity_boost": 0.75 },
            },
        )
        result = response.json()

    return {
        "audio_bytes": base64.b64decode(result["audio_base64"]),
        "alignment": result.get("alignment", {}),
    }

def alignment_to_subtitles(alignment: dict, audio_id: str, max_chars: int = 30) -> list:
    """내레이션 타임스탬프 → 자막 (오디오 내 상대 시간)"""
    chars = alignment.get("characters", [])
    starts = alignment.get("character_start_times_seconds", [])
    ends = alignment.get("character_end_times_seconds", [])

    cues = []
    current_text, current_start = "", 0

    for i, char in enumerate(chars):
        if not current_text:
            current_start = starts[i]
        current_text += char

        if char in ".?!。？！" or (len(current_text) >= max_chars and char == " ") or i == len(chars) - 1:
            st = int(current_start * 1000)
            et = int(ends[i] * 1000)
            cues.append({
                "linkedAudioId": audio_id,
                "startTime": st,
                "endTime": et,
                "text": current_text.strip(),
                "_originalStartTime": st,
                "_originalEndTime": et,
            })
            current_text = ""
    return cues

async def get_audio_duration(path: str) -> float:
    proc = await asyncio.create_subprocess_exec(
        "ffprobe", "-v", "quiet", "-show_entries", "format=duration",
        "-of", "csv=p=0", path, stdout=asyncio.subprocess.PIPE
    )
    stdout, _ = await proc.communicate()
    return float(stdout.decode().strip()) * 1000

async def generate_slide_narrations(project_id: str, slides: list) -> dict:
    """
    슬라이드별 내레이션 생성.
    음성 있는 슬라이드: duration = 음성 길이 + 800ms
    음성 없는 슬라이드: duration 그대로 유지 (수동 설정값)
    """
    audio_tracks = []
    subtitle_cues = []
    cumulative_time = 0
    tts_dir = f"data/media/{project_id}/tts"
    os.makedirs(tts_dir, exist_ok=True)

    for i, slide in enumerate(slides):
        script_text = slide.get("notes", "")

        if not script_text:
            # ⭐ 음성 없는 슬라이드 (목차, 인트로 등) — duration 그대로
            cumulative_time += slide["duration"]
            continue

        # 캐시 체크
        text_hash = _text_hash(script_text)
        audio_path = f"{tts_dir}/slide_{i}_{text_hash}.mp3"
        align_path = f"{tts_dir}/slide_{i}_{text_hash}.json"

        if os.path.exists(audio_path) and os.path.exists(align_path):
            with open(align_path) as f:
                alignment = json.load(f)
        else:
            result = await generate_narration(script_text)
            with open(audio_path, "wb") as f:
                f.write(result["audio_bytes"])
            with open(align_path, "w") as f:
                json.dump(result["alignment"], f)
            alignment = result["alignment"]

        audio_duration = await get_audio_duration(audio_path)
        audio_id = f"tts-{slide['id']}"

        # 자막 추출 (오디오 내 상대 시간)
        cues = alignment_to_subtitles(alignment, audio_id)
        # 타임라인 절대 시간으로 변환
        for cue in cues:
            cue["startTime"] += int(cumulative_time)
            cue["endTime"] += int(cumulative_time)
        subtitle_cues.extend(cues)

        audio_tracks.append({
            "id": audio_id,
            "type": "tts",
            "src": audio_path,
            "linkedSlideId": slide["id"],
            "startTime": int(cumulative_time),
            "duration": int(audio_duration),
            "volume": 1.0,
            "_originalStartTime": int(cumulative_time),
        })

        # 음성 있는 슬라이드: duration = 음성 + 여유
        slide["duration"] = int(audio_duration) + 800
        cumulative_time += slide["duration"]

    return {
        "audio_tracks": audio_tracks,
        "subtitle_cues": subtitle_cues,
        "total_duration": int(cumulative_time),
        "updated_slides": slides,
    }
```

```
시나리오 1: 일반적인 흐름
  1. 스크립트 작성 → AI가 슬라이드 생성
  2. [내레이션 생성] → 음성 생성 → 자막 자동 → 슬라이드 길이 자동
  3. 프리뷰 확인 → 내보내기

시나리오 2: 중간에 목차/인트로 슬라이드 삽입
  1. 내레이션 이미 생성된 상태
  2. 슬라이드 2번과 3번 사이에 [목차 슬라이드 추가] (음성 없음, 5초)
  3. 시스템이 자동으로:
     → 목차 뒤의 모든 오디오 startTime이 5초 밀림
     → 자막 startTime도 5초 밀림
     → 음성 파일 자체는 재생성 안 함!
     → 타임라인 UI 즉시 반영
  4. 프리뷰: 목차 구간은 조용히 → 다음 슬라이드에서 음성 재개

시나리오 3: 슬라이드 순서 변경
  1. 드래그로 슬라이드 순서 바꿈
  2. 연결된 음성/자막이 슬라이드 따라서 자동 이동
  3. 음성 재생성 없음

시나리오 4: 스크립트 수정
  1. 특정 슬라이드의 스크립트 텍스트 변경
  2. 해당 슬라이드의 음성만 재생성 (나머지 유지)
  3. 자막/duration 자동 재계산 → 뒤쪽 startTime 자동 밀림/당김
```

# media_service.py
from PIL import Image
import os

async def process_upload(file, media_id: str) -> dict:
    """업로드된 이미지 자동 처리"""
    media_dir = f"data/media/{media_id}"
    os.makedirs(media_dir, exist_ok=True)

    # 원본 저장
    original_path = f"{media_dir}/original{ext}"
    with open(original_path, "wb") as f:
        f.write(await file.read())

    img = Image.open(original_path)
    orig_w, orig_h = img.size
    aspect_ratio = orig_w / orig_h

    # 썸네일 (에디터 목록용)
    thumb = img.copy()
    thumb.thumbnail((360, 360))
    thumb.save(f"{media_dir}/thumbnail.webp", "WEBP", quality=80)

    # 프리뷰 (편집 화면용, 1920px)
    if orig_w > 1920:
        preview = img.resize((1920, int(1920 / aspect_ratio)), Image.LANCZOS)
    else:
        preview = img.copy()
    preview.save(f"{media_dir}/preview.webp", "WEBP", quality=90)

    # 4K 렌더링 시에는 원본 사용
    return {
        "id": media_id,
        "url": f"/api/media/{media_id}/original",
        "thumbnail": f"/api/media/{media_id}/thumbnail",
        "preview": f"/api/media/{media_id}/preview",
        "width": orig_w,
        "height": orig_h,
        "aspectRatio": aspect_ratio,
    }

function renderImage(element, canvasWidth, canvasHeight) {
  const pos = toPixel(element.position, canvasWidth, canvasHeight);
  const img = document.createElement('img');

  // 해상도에 따라 소스 선택
  if (canvasWidth >= 3840) {

  } else {

  img.style.position = 'absolute';
  img.style.left = pos.x + 'px';
  img.style.top = pos.y + 'px';
  img.style.width = pos.width + 'px';
  img.style.height = pos.height + 'px';
  img.style.objectFit = element.style?.objectFit || 'cover';
  return img;
}

## 11. 환경 변수 & API 키 관리

> **상세 키 값은 `환경변수_API키_가이드.md` 참조.**

### 11-1. 핵심 규칙

- 모든 API 키는 **`.env` 파일에만** 저장
- `.env`는 **`.gitignore`에 반드시 포함**
- 코드에서는 **`os.getenv()`** (Python) / **`process.env`** (Node.js)로만 접근
- 프론트엔드에 API 키 절대 노출 금지 (`VITE_` 접두사 붙이지 말 것)
- `.env.example` 파일을 Git에 포함하여 변수명만 공유

### 11-2. 환경 변수 목록

| 변수명 | 서비스 | 사용처 |
|--------|--------|--------|
| `ANTHROPIC_API_KEY` | Claude API | 슬라이드 자동 생성, 채팅 수정 (ai_service.py) |
| `GOOGLE_AI_API_KEY` | Gemini API | 보조 AI (선택) |
| `ELEVENLABS_API_KEY` | ElevenLabs | TTS 음성 생성 (render_service.py) |
| `GITHUB_PAT` | GitHub | 저장소 접근, CI/CD |
| `HOST` | 서버 | FastAPI 바인딩 주소 (기본: 0.0.0.0) |
| `PORT` | 서버 | FastAPI 포트 (기본: 8000) |
| `DEBUG` | 서버 | 디버그 모드 (기본: true) |
| `DATA_DIR` | 파일시스템 | 데이터 루트 (기본: ./data) |
| `MEDIA_DIR` | 파일시스템 | 미디어 저장 (기본: ./data/media) |
| `RENDERS_DIR` | 파일시스템 | 렌더링 결과 (기본: ./data/renders) |
| `DB_PATH` | SQLite | DB 경로 (기본: ./data/db.sqlite) |

### 11-3. 코드 패턴

**Python (FastAPI) — 모든 서비스 파일에서 이 패턴 사용:**

```python
from dotenv import load_dotenv
import os

load_dotenv()  # 프로젝트 루트의 .env 로드

# 필수 키 — 없으면 서버 시작 시 에러
ANTHROPIC_API_KEY = os.getenv("ANTHROPIC_API_KEY")
ELEVENLABS_API_KEY = os.getenv("ELEVENLABS_API_KEY")

if not ANTHROPIC_API_KEY:
    raise ValueError("ANTHROPIC_API_KEY가 .env에 설정되지 않았습니다")
```

**Node.js (render-worker):**

```javascript
require('dotenv').config({ path: '../.env' }); // 프로젝트 루트 참조
```

**React (Vite) — API base URL만 노출:**

```env
VITE_API_BASE_URL=http://localhost:8000/api
VITE_WS_BASE_URL=ws://localhost:8000/ws
```
